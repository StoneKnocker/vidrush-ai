# Code Review: Seedance 视频生成工作区
##  并行上传文件路径冲突

**文件**: `app/lib/r2/r2.client.ts` → `generateUploadFilePath`  
**关联**: `app/components/video-workspace/generation-form.tsx` 的 `uploadFiles`

`generateUploadFilePath` 用 `Date.now()` 生成路径：

```ts
return `uploads/${userId}/${timestamp}.${ext}`;
```

`uploadFiles` 已改成 `Promise.all` 并行上传。多个文件可能在同一毫秒拿到相同路径，后上传的文件会覆盖先上传到 R2 的对象，导致多个 asset 指向同一个 URL，引用被串改或丢失。

**建议**: 路径加随机片段，例如：

```ts
return `uploads/${userId}/${timestamp}-${crypto.randomUUID()}.${ext}`;
```

##  Portrait 选中状态与 asset 列表不同步

**文件**: `app/components/video-workspace/generation-form.tsx` → `addPortraitAsset`

`addPortraitAsset` 在检查数量限制之前就调用了 `setSelectedPortrait(portrait)`：

```ts
const addPortraitAsset = (portrait: PortraitItem) => {
  setSelectedPortrait(portrait); // ← 先设置了
  setAssets((current) => {
    ...
    if (imageCount >= ASSET_LIMITS.image.maxFiles) {
      setFormError(...);
      return current; // ← 实际没加进去
    }
    ...
  });
};
```

当图片已达上限时，按钮 label 已经显示选中的国家名，但 assets 里并没有这张图，用户会看到“选中了但没用”的错位状态。

**建议**: 把 `setSelectedPortrait` 移到成功添加分支里，或者先判断限制再设置。

##  删除 Portrait asset 后选中状态未重置

**文件**: `app/components/video-workspace/generation-form.tsx` → `removeAsset`

`removeAsset` 只从 `assets` 里移除并 `revokeObjectURL`，但 `selectedPortrait` 不会被清除。结果是用户删掉了 portrait，按钮上还显示着原来的国家名。

**建议**: 在 `removeAsset` 里判断被删的是否是 portrait，如果是则 `setSelectedPortrait(null)`。
